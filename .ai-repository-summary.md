# AI Repository Summary

> This file provides a concise overview for AI agents and developers working with this codebase.

## Project Overview

**OS-APOW** (Open Source Agentic Process Orchestration Workflow) is a headless agentic orchestration platform that uses GitHub Issues as a work queue and executes tasks via DevContainers.

### Core Technology Stack

- **Language:** Python 3.12+
- **Framework:** FastAPI
- **Package Manager:** uv
- **HTTP Client:** httpx
- **Validation:** Pydantic
- **Testing:** pytest + pytest-asyncio
- **Containerization:** Docker

### Four Pillars Architecture

| Pillar | Component | File Location |
|--------|-----------|---------------|
| The Ear | Notifier | `src/os_apow/notifier/service.py` |
| The State | Queue | `src/os_apow/queue/github_queue.py` |
| The Brain | Sentinel | `src/os_apow/sentinel/orchestrator.py` |
| The Hands | Worker | `src/os_apow/worker/` |

## Important Patterns

### Distributed Locking (Assign-Then-Verify)

The sentinel uses an assign-then-verify pattern for task claiming:
1. Assign bot account to issue
2. Re-fetch issue to verify assignment
3. Only then update labels and proceed

### Secret Scrubbing

All output posted to GitHub is scrubbed using regex patterns in `work_item.py` to prevent credential leakage.

### Heartbeat Pattern

Long-running tasks post periodic heartbeat comments to keep observers informed.
